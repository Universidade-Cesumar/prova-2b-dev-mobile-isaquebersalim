# Sugestao de postagem para LinkedIn

Hoje finalizei a terceira sprint de um aplicativo mobile de controle de almoxarifado para o contexto da Enfermagem.

O projeto nasceu de um problema real: substituir o controle manual em planilha por uma solucao mobile mais pratica para registrar materiais, acompanhar estoque e realizar baixas diretamente na lista.

Durante as sprints, desenvolvi:

- Listagem dinamica consumindo uma MockAPI.
- Cadastro de novos materiais com React Native e Expo.
- Baixa rapida de estoque com validacao para impedir saldo negativo.
- Exclusao com confirmacao.
- Filtro de pesquisa em tempo real.
- Dashboard com total de itens, unidades e itens criticos.
- Alerta visual para estoque abaixo de 10 unidades.
- Tratamento de erros de conexao para manter a interface estavel.
- Testes automatizados com Jest.
- Documentacao no README com instrucoes de execucao via Expo Go.

Foi uma experiencia importante para praticar um fluxo mais proximo do mercado: entender requisitos, dividir entregas por sprint, versionar com commits semanticos, consumir API REST e validar regras de negocio.

Tecnologias usadas: React Native, Expo, JavaScript, Hooks, fetch, MockAPI e Jest.
